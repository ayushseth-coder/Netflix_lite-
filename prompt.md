# Sprint-8 Reference Prompts (Information Only)

## Purpose of This File
This file documents the **thinking process and reasoning prompts** used while designing the Sprint-8 project.

These prompts are included for:
- Concept clarity
- Viva & interview preparation
- Architectural justification

⚠️ These prompts are **NOT meant for direct copy-paste execution**.

---

## Architecture & Design Prompts
- How should a Vanilla JavaScript SPA be structured for scalability?
- Why should API logic, UI rendering, state, and utilities be separated?
- What problems occur when thousands of DOM nodes are rendered at once?

## Infinite Scroll Prompts
- Why is infinite scrolling preferred over pagination for large datasets?
- How does Intersection Observer improve performance?
- What happens if infinite scroll is not capped or controlled?

## Debouncing & API Optimization Prompts

- How does debouncing reduce server load and UI lag?

- What issues arise without proper state synchronization?
